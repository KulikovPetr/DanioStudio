DanioRecorder - программа для одновременной записи видеофайлов с восьми веб-камер.
При запуске, открывается окно программы с графическим интерфейсом и консоль.
Консоль используется для отслеживания состояния программы.

Настроенная по умолчанию, программа будет записывать видеофайлы с восьми камер с 9:00 до 21:00 длительностью 14 минут каждые 15 минут с частотой кадров 1кадр/сек.

Рекомендуется запускать программу от имени администратора, поскольку она создает файлы. По умолчанию, в операционной системе Windows 10, запрещено создавать файлы
без разрешения пользователя. Запуск от имени администратора позволяет преодолеть это ограничение.

Входные данные программы:
	Путь к папке сохранения файлов видеозаписей
	Строки настроек
Выходные данные программы: 
	Видеофайлы формата .avi

Интерфейс программы:
	1.первая строка(неактивная) - показывает текущее время компьютера(графическое удобство)
	2.Вторая строка - настройки записи в формате: чч:мм:сс начала записи, чч:мм:сс конца записи, количество камер, колво секунд до следующего старта записи, колво секунд до конца записи
	Например, нужно, чтобы 8 камер записывали с 9:00 до 21:00 каждые 15 минут видеозаписи по 14 минут. Такой настройке нужна инструкция: 9:00 21:00 8 900 840
	интервал в 60 секунд не обязателен, но желательно оставлять некоторое время "простоя", чтобы файлы успели сохраниться, а камеры - освободиться.
	3.Кнопка "Pause" служит для приостановки работы программы. Текущий цикл записи при этом не останавливается, а продолжается. Поэтому, перед тем, как
	провести технические работы(например, чистка аквариумов), стоит дождаться сообщения в консоли об остановке записи(окончания записи). Программа будет пропускать
	циклы начала записи, пока программа находится "на паузе".
	4.Кнопка "Resume" служит для возобновления записи после паузы. После нажатия на неё, программа дождется очередного времени начала записи и только после начнет
	записывать видеофайлы.
	5.Кнопка "Set" служит для установки настроек записи (активируется после нажатия кнопки "Folder")
	6.Кнопка "Start" служит для запуска программы с указанными настройками (активируется после нажатия кнопки "Set")
	7.Кнопка "Forced Start" является аналогичным кнопке "Start" началом записи, за исключением того, что запись начинается с момента нажатия, не дожидаясь временного интервала
	8.Кнопка "Quit" служит для выхода из программы по аналогии с кнопкой крестика
	9.Кнопка "Folder" служит для выбора папки, в которую будут сохраняться видеозаписи. В ней создадутся ещё 8 папок, в которые будут сохраняться видеофайлы соот
	ветствующих камер.
	10.Текстовые поля "Relay 1/2 ON/OFF" служат для установки времени включения/выключения реле*
	11.Кнопки "Relay 1/2 ON/OFF" служат для ручного включения/выключения реле*
	12.Черный квадрат служит для ручного определения номеров камер. При нажатии на текстовые поля "введите номера камер"(см п.13), на них будет подаваться
	изображение с камеры.
	13.Текстовые поля "введите номера камер" служат для ручного ввода номеров камер. Поскольку операционная система определяет usb-камеры по-своему, необходимо
	удостовериться, что все номера камер соответствуют порядку и/или изменить их, чтобы при нажатии на текстовое поле показывалась камера, соответствующая 
	своему порядковому номеру. В нашем случае был выставлен порядок камер 0 1 5 7 2 3 4 6.
	14.Кнопка "Save" служит для сохранения порядка камер в файле CamerasConfig.
	15.Чекбокс "Use relay" служит для включения управления реле*. Если оно не подключено или исполняемые файлы не лежат в нужном месте, могут возникать ошибки.
	16.Текстовое поле, содержащее путь к папке, содержащей программы для управления реле*

Во время "простоя" программы(не когда она на паузе, а после времени остановки записи и до старта записи(не 14/15 минутные интервалы, а 9:00/21:00 интервалы) 
стоит перезапустить программу, поскольку за время работы в консоли может сохраниться большое количество записей, занимающее много оперативной памяти.

Во время записи видео нельзя нажимать на текстовые поля камер(п.13), поскольку они "опрашивают" работающие видеокамеры и, соответственно, не получают доступ к ним. 
вызывается ошибка.

*для эксперимента используются usb-реле, они отвечают за включение света в установке. Настоятельно рекомендуется включать реле до записи и выключать 
после окончания записи, поскольку изменение света создает сильный шум на видеозаписи в первые 1-2 секунды.
Программное обеспечение для них используется и находится в открытом доступе на github: https://github.com/EssexPAL/usb-relay-hid

Использование:
	1.Запустить программу и проверить/изменить настройки времени записи(п.2),и, если используется реле таймеры включения и выключения реле(п.10), 
	чекбокс реле(п.15) и путь к реле(п.16)
	2.Проверить соответствие номеров камер (п.12 и п.13)
	3.Нажать кнопку "Folder" и выбрать папку, куда будут сохраняться видеофайлы
	4.Нажать кнопки "Set" и "Start".
	5.Когда на компьютере подойдет заданное время, реле переключатся и начнутся циклы записи. Во время работы программы, 
	каждые 15 минут будут создаваться 14-минутные видеофайлы в указанной директории.
		5.2 Если в процессе работы программы нужно будет приостановить запись, нужно нажать кнопку "Pause", дождаться сообщения в консоли о том, что 
		очередные видеофайлы созданы, и тогда можно приступать, к техническим работам с установкой. Когда они будут выполнены, нужно будет нажать кнопку
		"Resume", после чего программа дождется окончания 15-минутного(по умолчанию) интервала, и продолжит запись, о чем сообщит в консоль.
	6.Когда на компьютере подойдет время окончания времени записи, программа будет ждать начала следующего дня и нового времени включения,
	чтобы снова включить запись. Следом выключатся реле.
	7.После окончания времени работы программы настоятельно рекомендуется перезапустить программу(от имени администратора), чтобы очистить консоль.

Язык программирования: C#
Используемые библиотеки: System, Emgu.CV
