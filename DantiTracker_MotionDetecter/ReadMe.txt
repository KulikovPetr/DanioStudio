MotionDetecter - программа для анализа видеофайлов. 

Ссылка на видео с небольшой демонстрацией: https://www.youtube.com/watch?v=6GIXrkteJnk

Входные данные: видеофайлы формата .avi, три файла изображения формата .png того же разрешения, что и выбранные видеофайлы. 
Выходные данные: текстовые файлы для каждого видеофайла, состоящие из строк формата:
	номер_кадра колво_пикселей_общей_маски_дельта* колво_общей_маски_пикселей колво_пикселей_первой_маски_дельта* колво_пикселей_первой_маски колво_пикселей_второй_маски_дельта* колво_пикселей_второй маски
	*дельта - разница в пикселях между текущим кадром и предыдущим. Разница вычисляется применением оператора xor (исключающее или) для двух последующих кадров
	
	файл с именем первого файла и постфиксом All_analysis, содержащий сумму всех колонок каждого записанного файла.

Использование:
Для начала нужно нажать кнопку "выбрать файлы для чтения" и выбрать один или несколько видеофайлов(за раз)
Следом можно нажать кнопку "screenshot", чтобы сохранить изображение первого кадра видеозаписи для последующего создания маски в графическом редакторе(например, paint)
Далее следует выбрать три маски: global, mask1, mask2, нажатием на соответствующие кнопки. 
	global-маска используется, чтобы определить область, в которой будут обсчитываться данные
	маски 1 и 2 используются для задания суб-областей, в которых будут дополнительно обсчитываться данные(например верхняя и нижняя трети)
	Если маски 1 и 2 не нужны, их все равно нужно задать, но можно использовать тот же файл, что и для global-маски
Следом нужно нажать кнопку "start!" и дождаться, когда программа выдаст всплывающее оповещение о том, что она закночила работу.
Периодически можно поглядывать в консоль, там пишется прогресс работы: сколько файлов обсчитано и есть или нет ошибки*.
*в консоли могут возникать предупреждения в огромных количествах, но на работу программы они не сказываются.

Примечание: программа не оборудована "защитой от дурака", в ней нет защиты от неправильной последовательности команд, поврежденных файлов и несоответствия форматов.
программа может "вылетать" или "повисать", если один из выбранных файлов поврежден. 
Поврежденный файл можно определить, посмотрев на последний созданный текстовый файл. 
Он должен быть пустым(если видеофайл с таким же именем поврежден),
либо полностью верным(если поврежден следующий видеофайл). 
В таком случае стоит найти поврежденный видеофайл(по имени) и открыть его через любой видеопроигрыватель. Стоит просмотреть количество кадров в нем и, 
если оно меньше положенного, удалить. Такой файл обработать не получится, он неправильно записался.  


Язык программирования: python 3.10
Используемые библиотеки: tkinter, numpy, cv2
